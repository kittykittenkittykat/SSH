먼저 setuid 바이너리 suconnect의 설명을 읽어보면, localhost의 특정 포트에 연결해서 상대측에서 보내는 비밀번호를 확인해 다음 레벨의 비밀번호를 출력하거나 안하거나 입니다.(오답을 입력했을때 뭘 할지는 안해봐서 모릅니다)
문제의 NOTE에 나와있듯이 이 '상대측'의 입장을 제가 합니다. 즉, 3333번 포트에서 listen, 즉 연결 요청을 기다리는 서버 역할을 합니다.
그리고 연결 요청을 기다리는 서버역할을 하는 이 셸은 그대로 두고 셸을 하나 더 실행시켜서 클라이언트 역할을 하는 suconnect 의 사용법대로 방금 열어둔 3333포트에 접속합니다.
이러면 두 셸 창이 생겼고 서로 연결이 되었습니다.
이제 suconnect의 설명에 따라 서버 역할을 하는 먼저 켜둔 창에 bandit20의 비밀번호를 입력해주면, 클라이언트 역할을 했던 나중에 킨 창에 비밀번호가 일치한다는 문구가 뜨면서 다음 레벨의 비밀번호를 전송해줍니다.
